keyDecisions:
  - decision: "Protocol Buffers for V2X message serialization"
    reasoning: "Binary serialization reduces message size by ~60% compared to JSON, keeping per-message latency within the 100ms budget. The schema registry also enforces contract compatibility between the vehicle ECU and the infrastructure backend."
    alternatives:
      - "JSON over HTTP/REST"
      - "Custom binary protocol"
  - decision: "Kafka as the V2X data backbone"
    reasoning: "Kafka's partitioned log provides replay capability for debugging and field test analysis. Multiple consumers (real-time dashboard, data lake, alerting) can independently subscribe without coupling."
    alternatives:
      - "Direct TCP socket communication"
      - "MQTT for IoT messaging"
  - decision: "Stateless middleware with Redis for session state"
    reasoning: "Stateless design allows horizontal scaling during high-traffic field tests. Redis stores in-flight message state with TTL to handle vehicle reconnects gracefully."
    alternatives:
      - "Stateful middleware (embedded session)"
      - "Database-backed session state"
