duration: "1개월"

outcomeSummary: "홈페이지 전체 리소스 용량을 57.9MB → 15.3MB로 줄여 클라우드 배포 플랜 초과 비용 제거"
overview: "클라우드 호스팅 플랜의 스토리지·전송 한도를 초과하여 매월 예상치 못한 초과 요금이 발생하는 원인을 파악하고 해결했습니다. 리소스 최적화 작업으로 시각적 품질 저하 없이 총 에셋 크기를 74% 줄였습니다."
problem: "회사 홈페이지가 매월 클라우드 배포 초과 요금을 지속적으로 발생시키고 있었습니다. 조사 결과, 배포 번들에 최적화되지 않은 이미지 에셋, 여러 CDN 소스에서 중복 로딩되는 웹폰트, 프로덕션에 포함되지 않아야 할 빌드 아티팩트가 포함되어 있었습니다. 총 에셋 크기가 57.9MB로, 일반적인 마케팅 사이트에서 허용 가능한 수준을 훨씬 초과한 상태였습니다."
constraints:
  - "이미지 시각 품질 유지 (인지 가능한 화질 저하 없음)"
  - "페이지 디자인 및 콘텐츠 변경 없음"
  - "플랜 업그레이드를 피하기 위해 기존 클라우드 배포 플랜 내에서 해결"
  - "지속되는 비용을 최소화하기 위해 1개월 내 완료"
approach: "Chrome DevTools와 번들 분석기를 사용하여 전체 배포 번들을 감사했습니다. 주요 원인 세 가지를 파악했습니다: 최적화 없이 서비스되는 대형 원본 이미지(~40MB), 여러 CDN 소스에서 중복 로딩되는 웹폰트(~8MB), 축소화되지 않은 CSS/JS 빌드 아티팩트. 이미지 WebP 변환, 폰트 소스 통합, 자동화된 크기 검사가 포함된 빌드 파이프라인 최적화를 적용했습니다."
keyDecisions:
  - decision: "모든 이미지 WebP 변환"
    reasoning: "WebP는 동등한 시각 품질에서 JPEG/PNG 대비 25~35% 더 작은 파일 크기를 제공합니다. 브라우저 지원율이 97% 이상이므로 폴백 복잡성이 최소화되고 크기 절감은 즉각적입니다."
    alternatives:
      - "AVIF (더 작지만 브라우저 지원율 낮음)"
      - "공격적인 JPEG 압축 (눈에 띄는 화질 저하)"
  - decision: "여러 CDN 소스 대신 폰트 셀프 호스팅"
    reasoning: "사이트가 누적된 설정 변경으로 인해 동일한 폰트 패밀리를 3개의 CDN 소스(Google Fonts, Adobe Fonts, 로컬 복사본)에서 로딩하고 있었습니다. 단일 셀프 호스팅으로 통합하여 중복을 제거하고 외부 DNS 조회 3건을 없앴습니다."
    alternatives:
      - "시스템 폰트로 되돌리기 (디자인 변경 필요)"
      - "CDN 레벨에서 중복 제거 유지"
  - decision: "빌드 파이프라인에 최적화 자동화"
    reasoning: "수동 일회성 정리는 새 콘텐츠가 추가되면 다시 회귀합니다. CI에 이미지 최적화(sharp)와 에셋 크기 한도 검사를 추가하면 향후 점진적 비대화를 방지합니다."
    alternatives:
      - "CI 강제화 없이 일회성 수동 정리"
      - "빌드 타임 게이트 대신 정기 감사"
techStack:
  - "Node.js / sharp (이미지 최적화)"
  - "WebP"
  - "Webpack / Vite (빌드 최적화)"
  - "GitHub Actions (CI/CD)"

    - label: "페이지 로딩 시간"

  qualitative: "홈페이지가 기존 클라우드 배포 플랜 한도 내에서 여유 있게 운영되어 예상치 못한 월별 요금이 없어졌습니다. 빌드 파이프라인 자동화 검사로 회귀를 방지합니다. 부수적으로 페이지 로딩 성능이 크게 개선되어 사용자 경험도 향상됐습니다."
learnings:
  - "배포 감사를 해보면 누적된 '임시' 설정 변경으로 인한 낭비가 드러남 — 정기적인 감사 프로세스가 점진적 비대화를 예방"
  - "이미지 에셋은 웹 번들 크기에서 거의 항상 지배적인 요소 — 배포 크기 문제 진단 시 여기서 시작"
  - "빌드 파이프라인을 수정하는 것이 일회성 정리보다 가치 있음 — 회귀할 수 있는 최적화는 최적화가 아님"
  - "클라우드 비용 초과는 종종 기술적 문제만이 아닌 모니터링 부재의 신호 — 해결책은 최적화만큼이나 관찰 가능성에 있음"
