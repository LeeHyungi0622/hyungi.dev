overview: "스마트 플랜트(공장) 모니터링 플랫폼의 백엔드 데이터 파이프라인과 AI 추론 레이어를 개발했습니다. 시스템은 산업 설비에서 실시간 센서 데이터를 수집하고, ML 기반 이상 탐지를 실행하여 플랜트 운영자에게 예측 정비 권고사항을 제공합니다."
problem: "공장 설비에는 수백 개의 센서가 지속적으로 데이터 스트림을 생성하고 있었으나, 기존 모니터링은 임계값 기반에 불과했습니다. 명백한 고장만 감지할 수 있고 예측은 불가능했으며, 운영자들은 반응적으로만 대응할 수 있었습니다. 이로 인해 비계획 설비 다운타임이 발생하여 생산 일정이 차질을 빚고 정비 비용이 증가했습니다."
constraints:
  - "수백 개의 동시 센서 스트림을 데이터 손실 없이 처리해야 함"
  - "AI 추론은 운영자가 즉시 조치를 취할 수 있도록 500ms 내 완료"
  - "기존 SCADA 및 PLC 시스템과의 연동"
  - "보안 정책으로 인해 온프레미스 배포 (퍼블릭 클라우드 불가)"
  - "4개월 구현 기간"
approach: "Apache Kafka를 활용하여 엣지 게이트웨이로부터 센서 데이터를 수집하는 스트리밍 데이터 파이프라인을 구축했습니다. 각 센서 스트림을 독립적으로 처리하는 이상 탐지 및 잔여 유효 수명(RUL) 예측 전용 모델 서빙 마이크로서비스를 구현했습니다. 실시간 알림 피드와 정비 권고 카드가 포함된 운영자 대시보드를 개발했습니다."
keyDecisions:
  - decision: "엣지-클라우드 계층형 처리 구조"
    reasoning: "고주파 센서 데이터(1000Hz+)는 백엔드로 스트리밍되기 전에 엣지 게이트웨이에서 사전 집계됩니다. 이상 탐지에 필요한 신호 충실도를 유지하면서 네트워크 대역폭을 ~90% 줄입니다."
    alternatives:
      - "센서에서 클라우드로 직접 스트리밍"
      - "센서에서 배치 폴링"
  - decision: "데이터 파이프라인과 모델 서빙 서비스 분리"
    reasoning: "분리를 통해 ML 팀이 데이터 수집 파이프라인을 건드리지 않고 새 모델 버전을 재학습하고 배포할 수 있습니다. 파이프라인 다운타임 없이 블루-그린 모델 배포가 가능합니다."
    alternatives:
      - "수집 파이프라인 내 추론 내장"
      - "스케줄 배치 ML 작업"
  - decision: "온프레미스 Kubernetes 클러스터"
    reasoning: "공장 보안 정책상 운영 데이터의 클라우드 배포가 금지됩니다. Kubernetes는 온프레미스에서도 동일한 오케스트레이션 기능과 자동화된 롤아웃 및 스케일링을 제공합니다."
    alternatives:
      - "Docker Compose (단순하지만 확장성 제한)"
      - "베어메탈 프로세스 관리"

learnings:
